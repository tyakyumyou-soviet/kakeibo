# Firestoreインデックス作成手順

## 現在の状況

アプリは正常に起動していますが、Firestoreのクエリに必要なインデックスがまだ作成されていません。

![初期状態のスクリーンショット](/Users/tyakyumyou/.gemini/antigravity/brain/5b24b845-ea80-4b65-80fb-500e9f3046cc/initial_app_view_1764346575018.png)

## エラー内容

```
支出読み込みエラー: FirebaseError: The query requires an index.
```

これは、`expenses`コレクションで`yearMonth`と`date`でソートするためのインデックスが必要という意味です。

## 解決方法

### 方法1: 自動生成リンクを使用(最も簡単!)

1. ブラウザの開発者ツール(F12)のコンソールタブを開く
2. エラーメッセージに表示されているURLをクリック
3. Firebaseコンソールが開き、インデックス作成画面が表示されます
4. 「作成」ボタンをクリック
5. インデックスの構築が完了するまで待つ(数分かかることがあります)

### 方法2: 手動でインデックスを作成

1. [Firebase Console](https://console.firebase.google.com/)を開く
2. プロジェクト「kakeibo-app-47a67」を選択
3. 左メニューから「Firestore Database」を選択
4. 「インデックス」タブをクリック
5. 「複合インデックスを追加」をクリック
6. 以下の設定で作成:
   - コレクション: `expenses`
   - フィールド1: `yearMonth` / 昇順
   - フィールド2: `date` / 降順
7. 「作成」をクリック

## インデックス作成後

インデックスの構築が完了したら(ステータスが「有効」になったら):

1. ブラウザで http://localhost:8000 をリロード(F5 または Cmd+R)
2. エラーが消えて、正常にアプリが動作することを確認

## 次のステップ

インデックス作成後、以下の機能をテストしてください:

- ✅ クレジットカードの追加
- ✅ 支出の入力
- ✅ 集計の確認
- ✅ 月の切り替え
- ✅ デバッグモード
